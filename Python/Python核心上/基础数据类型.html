  • 当括号前的内容比较短则采用隐式续行， 即元素垂直对齐于圆括号、方括号或花括号

悬挂缩进

  • 当括号前面的内容比较长则采用悬挂缩进，即在续行中再缩进一级即可，无须对齐括号，但我们要注意第一行不应该包含参数 ，并且应使用进一步的缩进来清除的区分自己作为延续行
python

        var_one, var_two, var_three,
        var_four): 
    print(var_one)
 
# 以下方式是不正确的，与前面代码一区分就很明显

# 第二行并没有和分界符对齐
foo = long_function_name(var_one, var_two,
    var_three, var_four)

# 没有再进行一级缩进同最后的代码做区分

注释 ​

在 Python3 中，注释不会影响程序的执行，但是会使代码更易于阅读和理解

Python中的注释有单行注释和多行注释

单行注释

以#开头，#右边的所有东西都被当做说明文字，而不是真正要执行的程序，只起到辅助说明作用

注意

为了保证代码的可读性，#后面建议先添加一个空格，然后再编写相应的说明文字

注释拓展说明

嵌套多行注释会导致语法错误

  • 原因是在 Python 中，多行注释是由三个单引号 ''' 或三个双引号 """ 来定义的，而且这种注释方式并不能嵌套使用。 当你开始一个多行注释块时，Python 会一直将后续的行都当作注释，直到遇到另一组三个单引号或三个双引号
  • 如果你需要在注释中包含嵌套结构，推荐使用单行注释（以#开头）而不是多行注释。 单行注释可以嵌套在多行注释中，而且不会引起语法错误

标识符由字母，数字和下划线组成，且不能以数字开头

标识符的命名方式 ​

1.小驼峰式命名法（lower camel case） ​

第一个单词以小写字母开始

第二个单词的首字母大写

例如：myBook、firstName

2.大驼峰式命名法（upper camel case） ​

每一个单字的首字母都采用大写字母

例如：FirstName、HelloWorld

3.下划线命名法 ​

用下划线_来连接所有的单词，比如hello_world

需要注意的几点

关于strip()方法

  • 所谓的空白字符有：空格(Space)，制表符(\t)，换行符(\n)，回车符(\r)，垂直制表符(\v)，空行等等
  • strip()方法，括号中你要去除的字符不存在，是不会报错的，只是返回原来的全部字符串
  • 例如案例中，为什么w字符存在，但是去除不了呢，因为w是位于中间位置，而我们strip是去除首尾位置的字符
  • 不一定首尾都要存在相同的字符才能取出,我们要保证位置是首尾就可以了,只有一个在首,或者只有一个在尾都可以去除

关于replace()方法

  • 同样的，如果replace()里面的字符不存在，不会报错，会返回默认字符串

关于split()方法

  • 首先是案例中是这样切割的
  • 连续多个空白字符会被当作一个识别
  • 返回的是一个列表类型
  • 因为返回的是一个列表类型数据,所以是不能进行链式调用,因为用点去进行链式调用,是要对字符串进行操作的,而不能对列表操作
  • 指定一个不存在的,则会把默认值作为一个列表返回

关于join()方法

  • 括号中要传入的是序列

总结 ​

对于and逻辑运算符，当前后两个值都满足为真，优先会返回后面一个值(因为前面的值为真了，就会往下继续执行，看后面的值，如果也为真，则会返回后面的值)，如果前面的值不满足了，那么直接返回前面的值

对于or逻辑运算符，当前面的值已经为真了，则不需要再看后面的值了，因为前面为真，这个整体怎么样都为真，所以直接返回前面的值。如果前面的值为假，则还不能确定整体的值是真是假，所以还要看后面的值，此时就是返回后面的值

and的优先级大于or
